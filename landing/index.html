Built for the messy middle of real agent development.

AgentDiff keeps the core capture layer universal, then enriches what it sees when your stack exposes agents, tools, graph nodes, handoffs, and retrieval steps.

01HTTP

Universal capture first.

Intercept LLM calls at the transport layer across OpenAI, Anthropic, Gemini, Mistral, Bedrock, Cohere, Azure OpenAI, LiteLLM, or raw provider calls.

02Δ

Behavioral diffs that matter.

Compare agent invocation rates, tool usage, handoffs, retrieval overlap, model config, prompt shifts, latency, token usage, and output stability.

03git

Causal attribution.

Map each non-passing delta back to changed files, prompt constants, tool schemas, model params, and reachable code with deterministic rules.

04CI

Ship it in CI.

Fail pull requests when internal behavior drifts, post report artifacts, and keep WARN thresholds visible without blocking every stochastic change.

05DB

Queryable traces.

Store runs in SQLite or export JSONL so teams can inspect trajectories, search events, replay reports, and debug regressions after the build finishes.

06doc

Reports people read.

Markdown, HTML, JSON, and PR-ready summaries show traditional eval beside behavioral eval so silent regressions become obvious.

Framework-aware, never framework-locked

Multi-agent adapters without becoming a framework zoo.

AgentDiff should observe and explain multi-agent behavior. It should not orchestrate it. V1 adapters add semantic labels over the universal capture core: graph nodes, speakers, roles, tasks, tools, retrievers, handoffs, and guardrails.
